# 技术总览

## 1. 当前产品闭环

当前仓库已经形成这条实际可用链路：

1. 用户通过 React 前端登录、浏览媒体库、进入播放页或管理页
2. 管理员可通过后端 `/admin` 页面填写主机侧配置与登录密码
3. 后端 FastAPI 负责认证、导入编排、媒体处理、加密、同步和播放
4. 导入时从 Windows 本地路径读取源视频，使用 `FFprobe` 获取元数据
5. 后端按固定字节大小切片，并使用 AES-256-GCM 对每个分片加密
6. 本地保存分片元数据、manifest、封面与任务状态
7. 分片与远端 manifest 可写入本地 `mock` 存储或百度网盘
8. 浏览器播放时只请求 `/api/videos/{id}/stream`
9. 后端按需读取本地缓存、远端对象或源文件，并返回原始视频字节流

## 2. 技术栈

### 后端

- Python 3.12
- FastAPI
- Starlette SessionMiddleware
- SQLite
- `cryptography`
- `httpx`
- `FFmpeg` / `FFprobe`

### 前端

- React
- TypeScript
- Vite
- React Router
- TanStack Query

## 3. 当前部署模型

- 可信 Windows 主机运行后端服务
- 局域网内其他设备只访问网页与播放流
- 加密密钥、数据库、源文件路径、百度凭据都保留在主机侧
- 浏览器不承担解密，也不直接访问百度网盘

## 4. 后端模块分层

### `src/app/core/`

- 基础配置
- 路径解析
- 登录校验
- 内容密钥读写

### `src/app/repositories/`

- 对 SQLite 的薄封装
- 只处理表读写，不承担业务编排

### `src/app/media/`

- `FFprobe` 媒体探测
- `FFmpeg` 封面抽帧
- 分片切分
- AES-256-GCM 加解密
- HTTP Range 到分片范围的映射

### `src/app/storage/`

- 存储抽象接口
- 本地 `mock` 远端实现
- 百度 OpenAPI 适配实现

### `src/app/services/`

- 导入任务
- 播放回退与分片预取
- 设置与管理员配置
- 百度 OAuth
- manifest 写入与延迟同步
- 封面与缓存管理

## 5. 配置职责分离

当前配置分为两类：

### 运行设置

由前端 `/settings` 页面管理：

- 存储后端类型
- 百度根路径
- 缓存目录
- 缓存上限
- 上传并发
- 下载并发

### 管理员设置

由后端 `/admin` 页面管理：

- 播放回退下载并发
- 百度 App Key / Secret Key / Sign Key
- 百度 OAuth 回调地址
- 登录密码
- 会话密钥

说明：

- 百度凭据现在优先读取本地数据库保存值，环境变量保留为兼容回退
- `session_secret` 属于启动级配置，保存后需要重启服务才能完全生效

## 6. 播放数据来源顺序

`/api/videos/{id}/stream` 当前按以下顺序解析视频内容：

1. 本地加密分片缓存
2. 当前配置的远端存储对象
3. 主机上的原始源文件

这样可以同时满足：

- 本地首选播放
- 远端回退播放
- 在远端分片缺失或本地缓存不完整时尽量不中断

## 7. 当前实现重点

- 登录与会话已可用
- 导入、播放、设置、后台任务、封面、缓存链路已打通
- 推荐页、媒体库、详情页、播放页、管理页、设置页已切到 `frontend/`
- 后端 `/admin` 页面已覆盖首次接入所需的关键主机侧配置

## 8. 仍需注意的边界

- 百度首次 OAuth 授权码流程仍需要管理员人工完成
- 主机路径导入仍以可信 Windows 主机为前提
- 后端仍保留源文件回退，这是一种可用性优先的工程折中
