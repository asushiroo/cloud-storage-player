# 测试与样例数据

## 自动化测试范围

当前自动化测试覆盖：

- 登录页访问
- JSON 认证接口登录态
- 未登录重定向
- 登录成功 / 失败
- 退出登录
- 密码哈希校验
- 目录接口鉴权
- 空目录返回
- 目录 / 视频数据返回
- `folder_id` 过滤
- 设置仓储读写
- 导入接口鉴权
- 导入任务创建
- 导入任务列表 / 详情
- 导入缺失文件的失败路径
- 设置 API 默认值与更新
- 导入成功后封面文件可访问

当前还**没有**自动化覆盖 Vue 前端构建和页面行为，这部分后续应通过：

- `npm run build`
- 前端单元测试
- E2E 测试

来补齐。

## 样例视频策略

虽然运行环境里可以手工放测试视频，但自动化测试没有直接依赖这些外部文件。

原因：

1. CI 或新环境里不一定存在相同样例
2. 文件路径不稳定
3. 仓库测试应尽量自包含

所以当前 `tests/test_import_api.py` 会通过 `ffmpeg` 生成一个极小的 MP4 文件作为样例。

## 生成测试视频的方法

测试里使用类似命令：

```bash
ffmpeg -y -f lavfi -i color=c=black:s=160x90:d=1 -c:v libx264 -pix_fmt yuv420p demo.mp4
```

这个视频的特点：

- 很小
- 有视频流
- 足够让 `ffprobe` 返回时长和格式信息

## 手工验证建议

如果你已经在本机 `/tmp` 或项目内 `tmp/` 放了测试视频，可以在服务启动后手工验证：

1. 先登录
2. 调用 `POST /api/imports`
3. 再调用：
   - `GET /api/imports`
   - `GET /api/videos`
   - `GET /api/videos/{id}`
   - `GET /api/settings`

建议的请求体示例：

```json
{
  "source_path": "/absolute/path/to/demo.mp4",
  "title": "tmp sample"
}
```

例如如果你把测试文件放在项目根目录下的 `tmp/rieri.mp4`，就应传服务器看到的绝对路径，而不是浏览器本地路径。

## 当前测试与真实导入的差异

自动化测试只是验证：

- API 行为
- 任务状态流转
- 媒体探测集成
- 数据库存储
- 封面抽取后的静态文件可读

它并没有覆盖未来的：

- 百度上传
- 分片加密
- manifest 同步
- 回放 Range 流

所以通过当前测试，只能说明“本地导入最小切片可用”，不能说明最终系统已经完整。
