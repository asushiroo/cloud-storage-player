# 前后端分离架构

## 目标

项目现在采用：

- 后端：FastAPI + SQLite + 服务层/仓储层
- 前端：Vue 3 + TypeScript + Vite + Pinia + Vue Router

核心原则是：

1. 后端负责数据、认证、导入、存储、加密、回放
2. 前端负责页面、交互、状态展示、表单提交
3. 双方只通过稳定的 HTTP JSON API 协作

## 为什么切换到前后端分离

相较于继续把 UI 写在后端模板里，前后端分离更适合这个项目后续增长：

- 页面复杂度会继续上升
- 导入任务、设置页、播放页都有较强交互
- 未来可能出现更多状态轮询或实时刷新需求
- 前端和后端可以独立演进

## 当前边界

### 后端负责

- Session 认证
- API 鉴权
- 本地导入任务
- 媒体探测
- 封面抽取
- 数据库存储
- 未来的分片、加密、百度上传、流式播放

### 前端负责

- 登录页
- 媒体库页
- 视频详情页
- 设置页
- 导入入口和导入任务状态展示

### 明确不放到前端的逻辑

- Baidu API 调用
- AES-GCM 加解密
- 明文视频流处理
- Range 解密拼装
- 主机本地密钥管理

## 当前认证模型

当前仍然使用后端 Session Cookie。

前端通过这些接口交互：

- `GET /api/auth/session`
- `POST /api/auth/login`
- `POST /api/auth/logout`

这意味着：

- 前端不自己保存密码材料
- 后端继续掌握认证状态
- Vue 前端只消费“是否已登录”的结果

## CORS 与开发模式

由于前端 dev server 和后端 API server 在开发时是不同端口，所以后端需要允许指定前端来源访问。

当前后端通过配置项控制允许来源，并开启：

- `allow_credentials = true`

这样前端发出的带 Cookie 请求才能工作。

## 迁移策略

当前仓库里还保留了少量后端模板页。
但这些页面现在只视为：

- smoke 测试入口
- 临时调试入口
- 迁移过渡期兼容层

**新功能 UI 不再继续写到 `src/app/web/`。**

## 目录层面的分离

### 后端

- `src/app/`

### 前端

- `frontend/`

### 文档

- `docs/`

这种结构的优点是：

- Python 依赖和 Node 依赖彼此独立
- 后端测试和前端构建互不耦合
- 后续部署也能更灵活
